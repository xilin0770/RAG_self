# 教育知识库项目实战需求说明
## 1. 项目背景
面向教育培训场景，建设一个智能教育知识库系统。系统需要接入课程资料、项目文档、教学讲义、题库与题目等内容，并提供课程介绍、内容检索、题目问答和项目问答能力。

## 2. 项目目标与功能范围
**内容导入**
- 支持课程资料、项目文档、讲义、题库与题目等内容导入，导入过程支持任务状态追踪与元数据管理（课程名、项目名、章节名、题库名、来源文件名等）

**课程检索与介绍**
- 可按课程名、项目名、知识点等条件查找，返回课程列表、详情、章节结构、先修要求和项目实战内容

**文档检索**
- 可定位课程文档或项目文档片段，返回课程名、项目名、章节名、文件名等来源信息

**题库检索**
- 可查询题目详情，返回题目内容、题型、选项、答案与解析

**知识问答**
- 基于知识库内容生成答案，返回必要的引用信息

**交互能力**
- 支持单轮问答、多轮对话、流式输出和历史记录管理

**质量要求**
- 系统应具备基础异常处理能力，避免导入失败导致服务整体不可用；查询结果应尽量准确、可解释、可追溯，流式输出应具备较好的实时性；各模块应具备清晰分层，便于后续扩展

## 3. 角色说明
### 3.1 主要使用对象
- 学员：查询课程介绍、课程详情、项目资料、学习内容和题目信息
- 助教：辅助定位课程文档、回答常见问题、快速检索知识点与题目

### 3.2 协同角色
- 教学运营：管理课程资料、上传项目文档、校验知识库内容

## 4. 典型业务场景
### 4.1 课程介绍场景
当用户提出”有哪些 Python 相关课程””推荐一个适合入门的 RAG 课程”之类的问题时，系统应根据课程目录与课程简介返回课程列表、课程定位、适合人群和学习目标等内容。

### 4.2 课程详情场景
当用户指定某一课程时，系统应能够返回课程简介、适用人群、章节结构、先修要求和项目实战内容等详细信息。

### 4.3 课程文档检索场景
当用户询问某个知识点、项目步骤或配置方法时，系统应优先返回相关文档片段，并在需要时生成结构化回答。返回结果中应尽量包含课程名、项目名、章节名、文件名等来源信息。

### 4.4 题目检索场景
当用户提出”Python 有哪些练习题”或”查询函数相关的选择题”等问题时，系统应能定位到对应题库或具体题目，并返回题目内容、题型、选项、答案与解析。返回结果中应包含题库名称、题目编码、课程归属等来源信息。

## 5. 内容要求
知识库中的每条内容建议包含以下字段：

- 内容正文
- 内容类型：课程介绍 / 文档片段 / 项目资料 / 题目
- 课程名称
- 项目名称
- 章节名称
- 题库名称（题目类型内容）
- 题目编码（题目类型内容）
- 题型（题目类型内容：单选题、多选题、判断题、填空题、简答题等）
- 来源文件名
- 来源路径或资源链接

## 6. 后续扩展方向
- 增加课程后台管理与知识库管理页面
- 增加文档增量更新能力
- 增加权限控制与多租户支持
- 增加学习路径推荐、能力测评等教育能力
- 增加教学视频检索能力，支持基于视频字幕、讲稿或时间轴文本定位相关片段
- 视频检索结果可返回视频标题、所属课程、开始时间、结束时间和片段摘要等信息
- 增加多模态检索，支持图片、课件截图与视频画面联合检索

## 7. 技术栈

| 层次 | 技术选型 | 说明 |
|------|---------|------|
| Web 框架 | **FastAPI** | 异步支持，自动生成 OpenAPI 文档，原生 SSE 流式输出 |
| RAG 框架 | **LangChain** | 文档加载、分割、检索链、Prompt 管理 |
| 向量数据库 | **ChromaDB** | 轻量级嵌入运行，适合本地开发和中小规模场景 |
| 嵌入模型 | **sentence-transformers** | 本地运行，推荐 all-MiniLM-L6-v2 (英文) / shibing624/text2vec-base-chinese (中文) |
| LLM | **OpenAI 兼容 API** | 可对接 OpenAI / DeepSeek / 本地 Qwen3-8B 等 |
| 元数据存储 | **SQLite + SQLAlchemy** | 课程、题库、导入任务等结构化数据管理 |
| 数据校验 | **Pydantic v2** | 请求/响应模型与配置管理 |
| 异步任务 | **BackgroundTasks + asyncio** | 导入任务异步执行与状态追踪 |
| 文档解析 | **unstructured / PyPDF2 / python-docx** | 支持 PDF、Word、Markdown、TXT 等格式 |

> **配置管理**: 所有 API Key、模型名称、嵌入模型参数、数据库路径等敏感或可变配置统一通过 `.env` 文件管理，使用 `python-dotenv` + Pydantic Settings 加载。`.env.example` 提供模板，实际 `.env` 不提交到版本控制。
