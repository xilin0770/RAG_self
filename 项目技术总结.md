# 教育知识库 RAG 系统 — 项目技术总结

## 项目概述

一个面向教育/培训场景的**智能知识库 RAG（检索增强生成）系统**。支持导入课程资料、项目文档、讲义和题库，通过语义搜索和流式对话对知识内容进行检索、问答和会话管理。目标用户为学生（查询课程信息、项目资料、题目）和助教（定位文档、答疑、检索知识点）。

## 技术栈

| 层次 | 技术 | 说明 |
|------|------|------|
| **语言** | Python 3.12 | 使用 uv 包管理器 |
| **Web 框架** | FastAPI 0.136+ | 异步、自动 OpenAPI 文档、SSE 流式支持 |
| **RAG 框架** | LangChain 1.2+ | ChatOpenAI 客户端、文本分割器 |
| **向量数据库** | ChromaDB 1.5+ | 持久化存储、余弦相似度、本地部署 |
| **嵌入模型** | BGE-M3 | 通过 sentence-transformers 加载，1024 维向量，中英双语 |
| **大语言模型** | Qwen3.5-27B（SiliconFlow API） | OpenAI 兼容接口，temperature 0.1 |
| **关系数据库** | SQLite + SQLAlchemy 2.0+ | 8 张元数据表 |
| **文档解析** | PyPDF2 3.0+、python-docx 1.2+ | 支持 PDF、DOCX、Markdown、TXT |
| **数据校验** | Pydantic v2 | 请求/响应模型 + Settings 配置 |
| **异步任务** | FastAPI BackgroundTasks | 导入流水线异步执行 |
| **流式响应** | SSE（sse-starlette） | 问答实时 token 流式输出 |
| **配置管理** | python-dotenv + pydantic-settings | 环境变量集中管理 |
| **测试** | pytest 9.0+、httpx | 单元测试 + 集成测试 |

## 核心架构

### 数据模型（8 张 SQL 表）

- `courses` — 课程元数据
- `projects` — 课程项目
- `chapters` — 章节结构
- `document_fragments` — 文档片段，关联 ChromaDB chunk ID
- `questions` — 题库（单选/多选/判断/填空/简答）
- `import_tasks` — 异步导入任务状态跟踪
- `conversations` — 对话会话
- `messages` — 对话消息

### 文档导入流水线（5 阶段）

1. **解析**：原始文件 → 纯文本（按扩展名选择解析器）
2. **分割**：RecursiveCharacterTextSplitter（chunk_size=500, overlap=50）
3. **向量化**：BGE-M3 生成 1024 维归一化向量
4. **向量存储**：写入 ChromaDB（collection: "education_kb"，余弦相似度）
5. **元数据**：写入 SQLite document_fragments 表，含 chunk_id 回链

### RAG 问答流程

1. 用户问题 → BGE-M3 向量化
2. ChromaDB 向量检索（top_k=5，支持元数据过滤）
3. 检索结果 + 系统提示词 → 构建上下文
4. 发送至 LLM 生成回答 + 引文来源
5. 同步返回（POST /qa）或 SSE 流式输出（POST /qa/stream）

### API 接口（18 个路由，6 个路由模块）

| 模块 | 主要接口 |
|------|---------|
| 导入 | `POST /import`、`GET /import/{id}/status` |
| 课程 | `/courses` CRUD + 子资源（章节、项目） |
| 搜索 | `POST /search/documents` |
| 题库 | `/questions` CRUD + 筛选 |
| 问答 | `POST /qa`（同步）、`POST /qa/stream`（SSE） |
| 对话 | `/conversations` CRUD + 消息管理 |

### Web 控制台

单页 HTML/JS 应用（`/web`），功能包括：
- API 地址配置
- 文件上传与导入（含元数据字段）
- 导入任务状态查看（手动/自动轮询）
- SSE 流式问答，实时输出与引文展示
- 响应式布局，中文排版优化
